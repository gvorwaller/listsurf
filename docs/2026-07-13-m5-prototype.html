Try this (matches the acceptance gates)

  1. Feel check: click rows rapidly — selection is instant (the app today lags ~½s; Phase 1 fixes it to this).
  2. Capture run: press Return with nothing selected → type → Return → Return… (continuation) → Esc.
  3. Things flow: filter Remaining, select the top row, hit Space-Space-Space — rows vanish, selection advances, ends at “All Done!”.
  4. Rename: select a row, press Return — edit in place, Esc cancels. Also ⌘E and double-click.
  5. Structure: Tab/⇧Tab to indent/outdent, ⌘⌥↑↓ to move. Verify they hit the highlighted row (the app today hits the last row).
  6. Menus: right-click a row vs the ⋯ button vs the Item menu — same commands, hints shown where honest.
  7. Drag: hover a row's text (cursor becomes grab), drag among its siblings. Multi-select first → drag refuses.
  8. Checkbox anatomy: parents show tri-state (orange minus = mixed) and progress; click a parent's checkbox to check its whole branch.
  9. iOS: the add field appears at the insertion point; delete confirms; ⓘ opens Details (notes + quantity); long-press a row for its menu (incl. Add Above).

Real in the app, simplified here

  • iOS touch drag: committed to main 2026-07-12 (M4), passes the simulator UI test, but has never been in any TestFlight build — device behavior unverified. Not simulated here.
  • Swipe-to-delete: in the edit-mode code and works in the simulator; absent in check mode and unverified on a real device — device check is an M5 Phase 2 gate item.
  • Undo: the registration pipeline exists (macOS ⌘Z works); iOS has no undo affordance today — M5 adds a visible Undo to the iOS toolbar overflow.
  • Edit List sheet (title, notes, icon, color) is real in-app; the ✎ here just explains itself.
  • Browser reserves a few keys (⌘N, ⇧⌘N, sometimes ⌘K/⌘[) — ⌃ (Ctrl) works as a stand-in for ⌘ everywhere here.
