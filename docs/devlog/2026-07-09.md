# 2026-07-09 — Consolidation plan implemented (phases 1–6)

Implemented the full remediation plan from `docs/2026-07-09-critical-review-consolidation.md` (td-0ea840). Net **−228 lines** across 31 files while adding features and tests. Base commit: `9e840b4`.

## What changed, by phase

**1. Keyboard unification (macOS).** Removed the bare Return/Tab key equivalents from the Item menu — they intercepted typing in every text field and were the root cause of "edit in place doesn't work" and "command keys don't work". Indent/Outdent menu items now use ⌘]/⌘[; the editor owns Return/⇧Return/Tab/⇧Tab directly via `.onKeyPress` (handlers return `.ignored` when inactive). Deleted `MacOutlineTabKeyMonitor.swift` (the app-global NSEvent band-aid) entirely. Menu commands are now disabled while a rename/add field is active (`ListStore.isTextInputActive` gates `focusedCommandActions`).

**2. Selection & inline editing.** Native `List(selection:)` is the sole selection owner on macOS (arrow keys, ⌘-click/⇧-click multi-select now work natively); iOS keeps one tap gesture at row level. Removed the three stacked tap-gesture layers and the custom selection highlight. Double-click on macOS renames again (via `contextMenu(forSelectionType:primaryAction:)`), Escape cancels, click-away commits. The same click-away discipline applies to the add field: selecting elsewhere commits typed text or cancels an empty field. Add/rename/deletion-pending state moved into `ListStore` as shared presentation state; the `OutlineAddRequest` binding plumbing was deleted.

**3. Undo repair.** Redo works for the first time: re-registration now happens synchronously inside the undo handler (`MainActor.assumeIsolated`), which is the only way `NSUndoManager` converts it into a redo entry. Undo actions target the store itself (the `UndoProxy` global singleton is gone) and are removed on store teardown, so a closed list can't be mutated by a stale Undo menu. Undo also restores selection. Deleted the dead `TreeCommand.swift` (119 lines, zero production callers) and its four tests; added `ListStoreUndoTests` covering the live path (undo/redo round-trip, subtree restore, selection restore, teardown, cross-mode).

**4. Store unification & error contract.** `ListDetailView.syncListMetadata` is gone — AppStore is the single owner of list metadata (navigation title, Edit List, and the inspector read it directly; `ListStore.list` is `internal(set)`). Export/import are present-only (no more present-AND-throw; call sites can't swallow errors anymore). Every save failure's retry button now retries the operation that failed instead of a destructive "reload". `loadLists` assigns atomically; stale-ID archive/restore/duplicate refresh the library instead of silently no-oping; `AppErrorStore` queues instead of clobbering; `ListStore.load()` awaits the persistence tail so a retry can't resurrect stale data.

**5. Persistence & integrity.** `OutlineRepository` collapsed to `fetchItems` + `applyChanges(saving:deletingIDs:)` — one logical mutation is one transaction, with rollback as policy in every repository catch (including the shared view context, which previously kept poisoned pending changes after a failed save). `ListRepository.delete(id:)` (the item-orphaning footgun) is gone; `fetchLibraryArchive()` reads the whole library in one background-context transaction so exports are snapshot-consistent. `TreeEngine`: moveUp/moveDown use true midpoints (the ±0.5 arithmetic could tie positions and leap two slots — regression-tested with fractional positions); `insertChild` with a missing parent degrades to a root append instead of persisting an orphan; indent/outdent boundary cases are no-ops instead of throwing a lying `itemNotFound`.

**6. Surface consolidation.** New `ItemActionsMenu` is the single definition of item actions, consumed by the selection context menu, the toolbar menu, and the per-row ellipsis (shortcut hints only on selection-targeted surfaces, fixing the lying ⌘⌫ hint). One deletion confirmation dialog, driven by `store.pendingDeletionIDs`. Notes preview: "Off" restored in Settings (0 = note icon in row), `NotePreviewView` is a plain `lineLimit` Text (no more scroll-stealing per-row ScrollView, Dynamic Type safe). Help regenerated to match reality: all Mac shortcuts, "Archived Lists", Settings documented; section collapse persists via AppStorage. Archive badge replaced with count-in-title where `.badge` doesn't render. ⌘N during store-recovery no longer arms a phantom sheet. Removed: false "paste multiple lines" promise, `items.filter { _ in true }`, `listContextMenu` passthrough, `AppError.importPartial`, `repairOrphans` duplicate API, `PersistenceStack` dead semaphore.

## Hostile review round (CODEX2 via relay, 2026-07-09 evening)

CODEX2 reviewed the uncommitted tree and reported 6 findings; all were verified real and fixed:

1. **Persistence tail vs. list deletion race** → two-sided fix: `CoreDataOutlineRepository.applyChanges` now refuses (and logs) item saves whose list no longer exists, in the same transaction; and `AppStore` keeps a weak registry of issued ListStores and drains their queued writes before `deleteList` and `exportLibrary` (which also fixes exports missing moments-old edits).
2. **Mode switch stranded add/rename state** → `ListStore.isCheckMode` didSet clears `editingItemID`/`addPlacement` on entering check mode; editor `.onAppear` re-seeds a rename draft after view recreation.
3. **Add/rename not mutually exclusive; iOS accessory relocated drafts** → `beginAdding`/`beginEditing` now clear each other; a rename draft ended externally is committed (not discarded) via the editingItemID onChange; accessory Below/Child commit the current draft first, making the Help text true.
4. **Item-save failure retried with destructive reload** → `enqueuePersist` retry re-queues the same (changed, deletedIDs) mutation; in-memory edits survive transient failures.
5. **Click-away gaps in add flow** → selection changes including deselection now dismiss (commit-or-cancel) a pending add; the `selectionFromAddFlow` guard protects the continue-adding flow. (Clicking the already-selected row on macOS remains a no-op — accepted; Escape/Return available.)
6. **Boundary no-ops polluted undo** → `indent`/`outdent` skip undo registration and persistence when the engine returns the tree unchanged; `updateItemTitle/Notes/Quantity` gained same-value guards.

New regression tests: deleted-list write refusal (real Core Data), retry-requeues-mutation (flaky repo), add/rename mutual exclusion, check-mode state clearing, boundary-indent-registers-no-undo. Suite grew 107 → 112. Re-verified: swift test 112/112; macOS plan green (102 unit + 6 UI); iOS plan green (102 unit + 4 UI).
