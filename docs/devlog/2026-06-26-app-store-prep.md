## Attempted but unfinished

### Screenshot capture — editor and check mode views
- XcodeBuildMCP's `snapshot_ui` does not detect SwiftUI NavigationLink rows as tappable targets. Only the ellipsis action-menu buttons and unnamed secondary buttons are exposed.
- Attempted workarounds:
  - `tap` on text elements (e52 "Japan Trip Packing") — not a tappable target type.
  - `tap` on unnamed buttons (e55) near the row — tapped at wrong coordinates, no navigation.
  - AppleScript `click at {x, y}` in the Simulator window — did not reliably translate to touch events.
  - Installed `cliclick` (Homebrew) for precise mouse automation — worked once in a batch loop (y=350–460), successfully navigated into the list and showed the check-mode detail view. Subsequent single-click attempts to reproduce navigation were inconsistent.
  - When successfully inside the list, `snapshot_ui` detected expand chevrons (e35–e61) and check buttons but tapping them via xcodebuildmcp navigated backward instead of expanding sections.
- Root cause: the Simulator window's coordinate mapping between xcodebuildmcp element refs, AppleScript, cliclick, and actual touch targets is unreliable for SwiftUI NavigationLink rows.
